Resumen ejecutivo

El presente informe técnico describe el desarrollo e implementación de un modelo de scoring crediticio destinado a predecir el riesgo de incumplimiento de los clientes sobre un préstamo financiero. El modelo se basa en datos históricos y utiliza técnicas de aprendizaje automático para proporcionar una herramienta de apoyo a la toma de decisiones. Se obtiene como resultado una página web con una interfaz amigable para que los usuarios realicen la consulta.

Introducción

Objetivo del proyecto:

En un mundo en constante cambio y evolución financiera, la evaluación precisa del riesgo crediticio se vuelve esencial para garantizar la estabilidad y confiabilidad de las transacciones económicas. En este contexto, este proyecto se enfoca en desarrollar una aplicación web que emplea un modelo predictivo basado en regresión lineal para determinar la probabilidad de que una persona caiga en incumplimiento (default) en un préstamo financiero.

Las variables consideradas para este análisis son fundamentales para comprender la dinámica de los solicitantes de préstamos y su relación con el riesgo crediticio. Dichas variables incluyen la edad de la persona (person_age), su ingreso anual (person_income), la propiedad de vivienda (person_home_ownership), la duración de empleo (person_emp_length), el propósito del préstamo (loan_intent), la calificación del préstamo(loan_grade), el monto del préstamo (loan_amnt), la tasa de interés (loan_int_rate), el estado del préstamo (loan_status) y el porcentaje del ingreso destinado al préstamo (loan_percent_income).

A través de este proyecto, buscamos implementar un sistema confiable y preciso que permita a las instituciones financieras evaluar de manera más efectiva el riesgo crediticio asociado con los solicitantes de préstamos, mejorando así la toma de decisiones y la gestión de riesgos en el sector financiero.

Alcance del proyecto:

El proyecto abarca desde la exploración y preparación de los datos hasta la implementación de una página web y creación de un video promocional. Esto incluye la construcción del modelo de scoring crediticio y su despliegue en línea.

Exploración y Preprocesamiento (preparación) de Datos

Origen de los datos:

El conjunto de datos utilizado en este proyecto es el "Credit Risk Dataset", obtenido de Kaggle. Este conjunto de datos proporciona la información necesaria para entrenar y validar nuestro modelo predictivo. Se incluyen información detallada sobre los clientes como edad, ingreso anual, tipo de vivienda en la que habita, así como datos importantes acerca del préstamo, monto, intereses, calificación del préstamo entre otras.

Durante la fase de exploración y preprocesamiento de datos, se llevaron a cabo diversas acciones críticas para garantizar la calidad y relevancia de los datos antes de su posterior utilización en el modelo predictivo. Estas acciones incluyeron:

Limpieza de Datos

  • Eliminación de valores nulos: Se identificaron y eliminaron registros que contenían valores nulos en cualquiera de las variables. Esto es crucial para asegurar que los datos sean completos y confiables.
  • Eliminación de duplicados: Se detectaron y eliminaron registros duplicados que podrían haber introducido sesgos o errores en el análisis.
  • Tratamiento de valores atípicos: Se identificaron valores atípicos, como edades improbables (por ejemplo, mayores a 120 años), y se eliminaron.

Análisis Exploratorio de Datos (EDA)

  • Eliminación de valores nulos: Se identificaron y eliminaron registros que contenían valores nulos en cualquiera de las variables. Esto es crucial para asegurar que los datos sean completos y confiables.
  • Eliminación de duplicados: Se detectaron y eliminaron registros duplicados que podrían haber introducido sesgos o errores en el análisis.
  • Tratamiento de valores atípicos: Se identificaron valores atípicos, como edades improbables (por ejemplo, mayores a 120 años), y se eliminaron.

Ingeniería de Características

  • Evaluación de la relevancia de las variables:
    Se realizaron análisis para determinar qué variables tienen un impacto significativo en la variable objetivo,     loan_status. Esto permitió excluir variables irrelevantes o de bajo peso, relacionado con esto eliminamos algunos registros en rangos donde no tenían un peso relevante como personas con ingresos desfasados por así decirlo, y se eliminaron, lo anterior asegura que el modelo predictivo se construya sobre atributos significativos, permitiendo una predicción precisa del riesgo crediticio. A continuación, se detalla el proceso y los criterios utilizados para seleccionar y transformar las características.
  • Pruebas Estadísticas:
    Se aplicaron pruebas estadísticas como la prueba de chi cuadrado y la prueba de F-statistics ANOVA para validar la importancia de las variables categóricas y numéricas, respectivamente.
    La prueba de chi cuadrado confirmó la relevancia de las variables categóricas, permitiendo una selección basada en evidencia estadística sólida.
    La prueba de F-statistics ANOVA, junto con la consideración del p-value en relación con un nivel de significancia de α (alfa) establecido en 0.05, ayudó a determinar qué variables numéricas eran estadísticamente significativas para el modelo.
  • Correlaciones:
    Se analizaron las correlaciones entre las variables para comprender mejor las relaciones y posibles multicolinealidades, asegurando que las variables seleccionadas fueran independientes y aportaran información única al modelo.
    La combinación de estos enfoques permitió una selección precisa y fundamentada de las características relevantes para la construcción del modelo predictivo. Este proceso de selección de características es esencial para garantizar que el modelo final esté compuesto por variables significativas que realmente influyan en la predicción del riesgo crediticio, contribuyendo así a la efectividad y precisión del modelo como bien se mencionó anteriormente.

Codificación de Variables Categóricas

Las variables categóricas, como "home_ownership", "loan_intent", y "loan_grade", fueron codificadas.

El objetivo de este proceso fue asegurar que los datos estuvieran limpios, consistentes y preparados adecuadamente para la construcción y validación del modelo predictivo. Este enfoque contribuye significativamente a la precisión y confiabilidad del modelo.

Construcción del Modelo

En esta fase, se construyó y validó el modelo predictivo utilizando la variable objetivo "loan_status". La construcción del modelo implicó una cuidadosa manipulación de variables categóricas y continuas, así como la utilización de técnicas como el cálculo de la Weight of Evidence (WoE) para evaluar el impacto de las variables en el resultado final. Aquí se detallan los pasos fundamentales en la construcción del modelo:

  • División de Datos:
    Se dividió el conjunto de datos en dos partes: un conjunto de entrenamiento que comprende el 80% de los datos y un conjunto de prueba que representa el 20%. Esta división permite evaluar el rendimiento del modelo de manera adecuada.
  • Cálculo de WoE para Variables Categóricas:
    Se utilizó la función woe_ordered_discrete para calcular el WoE de las variables categóricas. El WoE nos proporciona información sobre cómo cada categoría contribuye al resultado final de la variable objetivo "loan_status".
    El WoE se calculó para cada categoría de las variables categóricas, generando valores positivos y negativos que indican la influencia de cada categoría en la variable objetivo.
  • Cálculo de WoE para Variables Continuas:
    Se empleó la función woe_ordered_continuous para calcular el WoE de las variables continuas por intervalos. Este cálculo nos brinda información sobre cómo cada intervalo de valores contribuye al resultado final de "loan_status", de forma similar a lo que hace woe_ordered_discrete.
    De manera similar al cálculo para variables categóricas, el WoE se calculó, pero por intervalos, generando valores que indican la influencia de cada intervalo en la variable objetivo.
  • Identificación y Limpieza de Valores Atípicos:
    Aprovechando los valores de WoE obtenidos para las variables continuas, se identificaron y abordaron valores atípicos. Esto implicó una limpieza cuidadosa para asegurar que los pesos de evidencia reflejen de manera precisa la contribución de los intervalos de valores.

Con estos pasos, se logró una comprensión profunda de cómo las variables afectan la variable objetivo y se realizó la preparación de los datos para su posterior modelado.

Finalmente en esta etapa, se implementó el modelo predictivo utilizando la técnica de regresión logística. El proceso se llevó a cabo de la siguiente manera:

Se definió un pipeline que incluye dos componentes principales uno es la transformación usando Weight of Evidence (WoE) a las variables predictoras para transformarlas en valores que reflejen su impacto en la variable objetivo "loan_status". Luego se utilizó la regresión logística como modelo de predicción, configurada para manejar desbalances de clases mediante el parámetro class_weight.

Posteriormente se usó Validación Cruzada (Cross-Validation) el objetivo de la validación cruzada fue evaluar el rendimiento del modelo y mitigar posibles problemas de sobreajuste.

Evaluación del Modelo

En esta etapa del proyecto, se llevó a cabo la evaluación del modelo construido para predecir el riesgo crediticio y determinar su efectividad en la clasificación de los préstamos. Se emplearon métricas fundamentales para este análisis, centrándose en la discriminación y precisión del modelo. Aquí se detallan los aspectos clave de la evaluación del modelo:

  • Área bajo la Curva ROC (AUC-ROC):
    La métrica del Área bajo la Curva ROC (AUC-ROC) es un indicador ampliamente utilizado que evalúa la capacidad de discriminación del modelo entre las dos clases, en este caso, los préstamos que caen en default y los que no.
    La AUC-ROC varía entre 0 y 1, donde un valor más cercano a 1 indica un mejor rendimiento de clasificación.
  • Curva de Precisión y Recall (PR):
    La curva PR es otra métrica importante que evalúa la precisión del modelo en la clasificación de la clase positiva (default) en comparación con la clasificación incorrecta.
    Representa la relación entre la precisión (positivos verdaderos sobre positivos verdaderos más falsos positivos) y el recall (positivos verdaderos sobre positivos verdaderos más falsos negativos).
  • Resultados y Interpretación:
    Los resultados obtenidos a partir de la AUC-ROC y la curva PR proporcionan información valiosa sobre la calidad del modelo en términos de predicción de riesgo crediticio.
    Un valor alto en la AUC-ROC indica una buena capacidad de discriminación entre las clases de préstamos, mientras que una curva PR con una alta área bajo la curva señala una alta precisión y recall en la clasificación de los préstamos en default.

La evaluación del modelo se centró en garantizar que el modelo tenga la capacidad de identificar de manera precisa aquellos préstamos que podrían caer en default, lo que es fundamental para la toma de decisiones financieras informadas.

Integración en la Aplicación Web

Para lograr la integración del modelo predictivo en la aplicación web, se sigue un proceso que implica recibir una matriz de pesos correspondientes a las variables y realizar un producto punto con la matriz transformada, obteniendo así los puntajes que se incorporan en un scorecard. A continuación, se detalla este proceso:

Función de Transformación (Transform)

La función de transformación, denominada transform, es fundamental en este proceso. Toma la entrada del usuario, la cual contiene los campos relevantes de la solicitud de préstamo, y realiza la transformación necesaria para obtener una matriz de unos y ceros que representan las características de la solicitud.

Para cada variable categórica y rango de valores, se asignan valores binarios (0 o 1) indicando la presencia o ausencia de cada categoría en la solicitud de préstamo. Por ejemplo, en el caso de la variable "grade" (calificación del préstamo), se asigna un 1 si la calificación es 'A', 'B', 'C', etc., y un 0 si no lo es. Esto se repite para cada variable relevante.

Matriz de Pesos y Producto Punto

Una vez que se ha obtenido la matriz transformada, esta se multiplica por una matriz de pesos (coeficientes) correspondientes a cada variable. El producto punto de estas matrices produce un conjunto de puntajes para cada variable en el scorecard. Cada puntaje representa la contribución de esa variable al riesgo crediticio.

Por ejemplo, si la matriz de pesos es [0.2, 0.3, -0.1, ...], entonces el producto punto de los valores transformados y esta matriz de pesos dará como resultado un puntaje que se utilizará en el scorecard.

Scorecard y Predicción

Estos puntajes calculados se incorporan en un scorecard, que es un sistema utilizado para determinar la probabilidad de incumplimiento de un préstamo. El puntaje total en el scorecard se calcula sumando estos puntajes individuales.

Finalmente, este puntaje total se utiliza para predecir la probabilidad de que la solicitud de préstamo resulte en un incumplimiento (default) o no.

Esta metodología permite la integración eficaz del modelo predictivo en la aplicación web, permitiendo evaluar rápidamente la probabilidad de incumplimiento de un préstamo en función de la información proporcionada por el usuario.

Implementación y Despliegue

La etapa de implementación y despliegue es crucial para llevar el modelo predictivo de riesgo crediticio a los usuarios finales a través de una interfaz web. A continuación, se proporcionan detalles sobre cómo se implementó y desplegó la aplicación web utilizando tecnologías como JavaScript (JS), React y Bootstrap:

Tecnologías Utilizadas:

  • JavaScript (JS): Es el lenguaje de programación fundamental para la interactividad en la web. Se utiliza para manipular y actualizar el DOM, así como para integrar y gestionar el modelo predictivo en la aplicación web.
  • React: Se emplea como la librería/framework para la interfaz de usuario (UI) y para seguir un patrón de diseño estructurado y basado en componentes en el desarrollo de la aplicación web.
  • Bootstrap: Es una librería que proporciona estilos predefinidos y componentes para crear una interfaz de usuario atractiva y con un diseño responsivo.

Desarrollo de la Interfaz de Usuario:

Se utiliza React para diseñar y desarrollar la interfaz de usuario de la aplicación web. React permite crear componentes reutilizables que representan diferentes partes de la interfaz, lo que facilita la organización y la manipulación eficiente de la UI.

Bootstrap se utiliza para aplicar estilos a la interfaz y garantizar que la aplicación tenga un diseño atractivo y amigable para el usuario. También proporciona componentes listos para usar que agilizan el desarrollo.

Integración del Modelo Predictivo:

La función de transformación (transform) que se explicó anteriormente se integra en la lógica de la aplicación utilizando JavaScript. Esta lógica se invoca cuando el usuario proporciona información relevante para la predicción en el formulario inicial.

El puntaje calculado se muestra en la interfaz de usuario para que este tenga una comprensión visual de la evaluación de riesgo crediticio.

De esta forma se garantiza que la aplicación web esté lista para su uso y sea accesible para los usuarios, permitiéndoles utilizar el modelo predictivo de riesgo crediticio de manera efectiva.

Aprendizaje técnico:

  • Iteración: el desarrollo de modelos es un proceso iterativo. Se aprende a medida que se experimenta y se obtienen resultados.
  • Interpretación de Modelos: la interpretación de los modelos es tan importante como su precisión. Comprender por qué el modelo toma ciertas decisiones es fundamental para confiar en su capacidad.

Conclusiones

El modelo de regresión lineal desarrollado proporciona una estimación precisa de la probabilidad de incumplimiento de un préstamo en función de las características del solicitante. Esta herramienta puede ser valiosa para las instituciones financieras al evaluar y mitigar los riesgos crediticios asociados con los préstamos.

La Weight of Evidence (WoE) se convierte en una herramienta esencial para cuantificar la relación entre las variables y la variable objetivo.
